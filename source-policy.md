# 信源治理（白名单 / 黑名单 / 评分）

## 三层信源评级

### Tier 1：一手原始源（最高可信度）

**这一层是 `[公开数据]` 标签的唯一来源**。

- 公司财报、招股书、年报、电话会议、产品发布会
- 政策原文、政府公告、监管文件
- 专利文档原文（含权利要求）
- 一手访谈（已获授权 + 可追溯）
- 用户原始反馈、调研数据

**MCP 对接**：EdgarTools (SEC) / Google Patents / FactSet（详见 [source-skills-registry.md](source-skills-registry.md)）

### Tier 2：二手权威源（中等可信度）

**这一层匹配 `[行业共识]` 标签，要求 ≥ 2 个 Tier 2 来源交叉佐证**。

- 行业头部第三方报告（IDC / Gartner / 艾瑞 / QuestMobile / 沙利文 / 头豹研究院）
- 主流财经媒体深度报道（财新 / 晚点 LatePost / 36 氪深度 / 钛媒体 PRO / Bloomberg / WSJ / FT）
- 头部券商研报
- 专家访谈库（Guidepoint）
- LinkedIn 高管 / 招聘数据
- 行业大会公开演讲（含 PPT / 现场录像）

**MCP 对接**：Guidepoint / LinkedIn / Brave Search / TheirStack

### Tier 3：三手聚合源（线索级，不作为关键数字依据）

**这一层匹配 `[推断]` 或 `[需验证]` 标签**。

- 自媒体加工二手报道（**必须能追溯到 Tier 1 / 2 一手才能用**）
- 社交媒体（Twitter / X / 雪球评论 / 知乎专栏 / 小红书）
- 行业群讨论
- 短视频（B 站 / 抖音 / YouTube 行业号）

**MCP 对接**：last30days / xAI X Search

## 黑名单（一律不纳入）

- ❌ 公关稿、软文、付费 PR（独立性 = 0）
- ❌ 匿名爆料、小道、无具体来源的传闻
- ❌ > 18 个月不更新的具体数字（市场结构数据 > 6 个月就要打折）
- ❌ 单一来源未交叉验证的关键数字（如"超过 XX 万用户"在只有一处出现时不引用）
- ❌ LLM 生成的"二手报告"（看到 ChatGPT / Claude 生成的研报二次包装一律拒绝）

## 标签使用规则（强制）

每条洞察必须带证据标签：

| 标签 | 用法 | 例子 |
|---|---|---|
| `[公开数据]` | Tier 1 一手 | 财报数字、政策原文、招股书 |
| `[行业共识]` | Tier 2 多源（≥ 2 个独立来源）交叉佐证 | 多家 Tier 2 都说"行业 CR3 ≈ 60%" |
| `[需验证]` | 有 Tier 2 / 3 来源但仅单源，或新鲜度待补 | LinkedIn 上看到的招聘信号 |
| `[推断]` | 无直接来源，从公开信息逻辑推导 | "估计研发投入占成本 40%" |

**报告强制底线**：全文 `[公开数据]` 标签 ≥ 5。低于这个红旗，**退回去补一手证据**，不允许靠 `[推断]` 充数。

## 使用者本地白名单扩展

路径：`~/.company-diagnosis/source-allowlist.local.md`

用来加你本地化信任的源，特别是 MCP 生态没覆盖的中文头部媒体。模板：

```markdown
# <使用者公司> 信源白名单（仅本地，不公开）

## Tier 1 信任列表（视同一手）
- <如果你有特殊一手渠道，比如已签约的行业数据合作方>

## Tier 2 中文媒体（信任）
- 财新（付费版可信度高，免费版次之）
- 晚点 LatePost
- 36 氪深度（标"深度"前缀的稿件；普通号外不算）
- 钛媒体 PRO
- 第一财经
- 经济观察网

## Tier 2 行业报告（信任）
- 艾瑞 / QuestMobile / 易观 / 头豹研究院

## Tier 2 中文公司数据
- 天眼查 / 企查查 / 启信宝（注册信息、股权穿透、司法、招投标）
- 雪球长文（认证大 V，非短评）

## 拉黑名单（个人观察累积）
- <你认为不可信的源，写明拉黑理由>
```

跑 PEST / 五力时如果信源命中本地 Tier 2 白名单，标 `[行业共识]`。

## 中国市场缺口提醒

MCP 生态对中文一手源覆盖差。当前**没有**可用 MCP：

| 类别 | 缺失 |
|---|---|
| 公司注册 | 天眼查 / 企查查 / 启信宝 |
| 政策聚合 | 工信部 / 网信办 / 国家发改委 / 各地政府 |
| 中文财经媒体 | 财新 / 36kr / 虎嗅 / 钛媒体 / 晚点 / 第一财经 |
| 中文股票数据 | 雪球 / 同花顺 / 东方财富 / 富途 |
| 中文专利 | 国家知识产权局 CNIPA |

**变通方案**：靠 WebSearch + Brave Search MCP 实时检索 + 使用者本地 `source-allowlist.local.md` 标可信中文源。详见 [source-skills-registry.md](source-skills-registry.md)。

## 自检清单（报告末尾）

报告附录中必须含一段"信源自检"：

```markdown
### 信源自检
- 全文 [公开数据] 标签数：N（要求 ≥ 5）
- 全文 [需验证] 标签数：N（汇总到附录的待验证清单）
- Tier 1 一手源覆盖到的框架：PEST / 五力 / VRIO / ...
- 未覆盖到的框架（仅靠 Tier 2/3）：...（需说明原因）
- 黑名单是否触碰：未触碰 / 触碰了哪一条（必须改）
```
