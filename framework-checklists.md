# 六框架检查清单（Per-Framework Cheatsheet）

每节列出：必查维度 → 输出格式 → 传递到 SWOT 的规则。

---

## PEST

**必查 4 维度**（PEST 标准版；如果客户行业涉及环保/法律重大议题再扩成 PESTEL 加 Environmental + Legal）：

| 维度 | 必查项 |
|---|---|
| **Political 政治/政策** | 行业准入、政府采购、监管动向、反垄断、产业政策、地方补贴、税收变动、跨境合规、信创/国产替代 |
| **Economic 经济** | 宏观/行业增速、利率、汇率、可支配收入、客户预算、企业 CAPEX 趋势、上下游景气度 |
| **Social 社会** | 人口结构、消费习惯、代际偏好、舆论/品牌情绪、ESG 关注度、用户使用习惯变迁 |
| **Technological 技术** | 关键技术成熟度、替代技术、AI/大模型影响、专利壁垒、研发投入趋势、底层平台演进 |

**输出格式**：每条因素一行
`(维度) 因素描述 | 影响方向(+/-) | 时间维度(短/中/长) | 强度(1-5) | [证据标签]`

例：`(P) 信创/国产替代政策强化 | + | 中 | 4 | [公开数据]`

**强度评分锚点**：
- 5 = 改变行业格局
- 4 = 显著利好/利空头部玩家
- 3 = 中等影响
- 2 = 局部影响
- 1 = 边际影响

**传递到 SWOT 的规则**：
- 方向 + 且 强度 ≥ 3 → Opportunities
- 方向 - 且 强度 ≥ 3 → Threats
- 强度 ≤ 2 的 → 放附录段不进 SWOT 主体

---

## 波特五力（Porter's Five Forces）

**5 力 + 必查项**：

| 力 | 必查项 |
|---|---|
| **行业内竞争 Rivalry** | 玩家数量、份额集中度 CR3/CR5/HHI、行业增速、退出壁垒、产品同质化度、价格战频率 |
| **供应商议价 Supplier Power** | 供应商集中度、转换成本、自主可控度、关键供应是否被卡脖子（如 GPU/EDA/光刻机） |
| **买方议价 Buyer Power** | 客户集中度、单笔金额占比、转换成本、信息透明度、定制化需求强度 |
| **新进入者 New Entrants** | 资本壁垒、技术壁垒、品牌壁垒、规模经济、政策准入、网络效应、客户切换成本 |
| **替代品 Substitutes** | 跨品类替代、技术换代（如 AI Native 替代 SaaS）、性价比对比、客户切换意愿 |

**每力评级 1-5**：
- 1 = very weak（对在位者最有利）
- 5 = very strong（对在位者最不利）

每力评级后附 1-3 条理由。

**行业总吸引力分数**：`30 − (5 力评分总和)`。分越高行业越值得在内卷。

**传递到 SWOT 的规则**：
- Rivalry / New Entrants / Substitutes 评分 ≥ 4 → Threats
- Buyer Power 评分 ≥ 4：议价压力 → Threats；客户依赖结构 → Weaknesses
- Supplier Power 评分 ≥ 4 且公司不能自主可控 → Threats 或 Weaknesses

## 麦肯锡 7S

**3 个 Hard S**（看得见、易调整）：
- **Strategy 战略**：未来 2-3 年明确目标、差异化定位、增长路径
- **Structure 结构**：组织架构、汇报关系、决策权分配、事业部/职能/矩阵
- **Systems 系统**：流程、IT、考核 OKR/KPI、复盘机制、信息流转

**4 个 Soft S**（软性、慢变量）：
- **Shared Values 共同价值观**：使命、文化、价值主张（**核心 S**，居中位置）
- **Style 领导风格**：高层管理风格、决策风格、放权 vs 集权
- **Staff 员工**：人才结构、能力梯队、规模、流失率
- **Skills 组织能力**：独特的**组织级**能力（不是个人技能；如"大规模分布式系统稳定性"、"地推铁军")

**每个 S 的输出**：1-3 句现状描述 + 1 条关键证据。

**对齐矩阵（7×7）**：

|  | Strat | Struct | Sys | Values | Style | Staff | Skills |
|---|---|---|---|---|---|---|---|
| Strat | — | ✓/○/✗ | ... |
| ... |

对每一对 (Si, Sj) 评估：
- ✓ 对齐：两者相互支撑
- ○ 中性：彼此无强关系
- ✗ 错配：相互拉扯（如 Strategy 要做 ToB 大客户但 Structure 仍按 ToC 流量打法分钱）

**错配清单**：把所有 ✗ 配对列出，每个错配 1-2 句解释为什么是错配 + 影响。

**传递到 SWOT 的规则**：
- 多个 ✗ 集中在同一 S 上 → 该 S 是 Weakness
- 高度 ✓ 集中（如 Strategy + Skills + Systems 三角对齐）→ Strength

## VRIO

**决策树（必须严格顺序，禁止并行打分）**：

```
对每个资源/能力 R：
  V (有价值吗？能帮公司利用机会或抵御威胁？)
    └─ N → 竞争劣势 Competitive Disadvantage
    └─ Y → R (稀缺吗？只有少数对手拥有？)
              └─ N → 竞争对等 Competitive Parity
              └─ Y → I (难模仿/替代吗？复制成本高？)
                        └─ N → 暂时优势 Temporary Advantage
                        └─ Y → O (公司组织能力配套利用了吗？)
                                  └─ N → 未利用优势 Unused Advantage
                                  └─ Y → 持续竞争优势 Sustained Competitive Advantage
```

**输出表格**（强制）：

| 资源/能力 | V | R | I | O | 结论 | 证据 |
|---|:-:|:-:|:-:|:-:|---|---|
| 例：超大组织通讯录 | Y | Y | Y | Y | 持续竞争优势 | [公开数据] |
| 例：通义大模型接入 | Y | N | — | — | 竞争对等 | [行业共识] |

**至少分析 5-10 个资源/能力**，覆盖：
- 核心技术 / IP / 专利
- 品牌 / 用户心智
- 客户网络 / 组织关系 / 数据资产
- 渠道 / 政企关系
- 关键人才 / 创始团队
- 文化 / 流程 / 经验积累
- 生态 / 合作方网络

**禁用词**：高、中、低（这是模糊评分，不是 VRIO 决策树）。

**传递到 SWOT 的规则**：
- 持续优势 + 暂时优势 → Strengths
- 未利用优势 → 放 Weaknesses 段（标注"潜在 Strength，组织未跟上"）
- 竞争劣势 → Weaknesses
- 竞争对等 → 不进 SWOT 主体，放附录

## 价值链（Porter Value Chain）

**主要活动（5）**：

| 标准活动 | 标准定义 | SaaS/数字业务改写 |
|---|---|---|
| **进货物流 Inbound Logistics** | 原材料采购、入库、检验 | 数据采集、研发原料、算力采买 |
| **运营 Operations** | 生产、加工、装配 | 产品研发、平台运维、内容生产 |
| **出货物流 Outbound Logistics** | 仓储、配送 | 产品交付、SaaS 订阅开通、API 接入 |
| **营销与销售 Marketing & Sales** | 广告、销售、渠道 | 增长营销、销售铁军、渠道分销、ISV 联营 |
| **服务 Service** | 售后、维修、保养 | 客户成功、技术支持、培训、文档 |

**支持活动（4）**：

| 活动 | 必查 |
|---|---|
| **公司基础设施 Firm Infrastructure** | 治理、财务、法务、合规、IR、战略管理 |
| **人力资源管理 HRM** | 招聘、培养、激励、保留、组织建设 |
| **技术开发 Technology Development** | 核心技术研发、IP 积累、平台底座、AI 能力 |
| **采购 Procurement** | 算力/带宽/IDC 采购、第三方组件、合作方采购、办公采购 |

**每活动填的内容**（强制）：

| 列 | 填什么 |
|---|---|
| 占总成本估算 % | 具体数 或 高/中/低 + 1 句理由 |
| 类型 | cost driver / differentiator / both |
| vs 主要竞品 | 领先 / 持平 / 落后 + 1 句理由 |
| 证据 | [公开数据] / [行业共识] / [推断] |

**核心问题**：钱花在哪？毛利从哪来？比竞品哪一段更强？

**传递到 SWOT 的规则**：
- differentiator + 领先 → Strength
- cost driver + 落后 → Weakness
- 关键活动被外部卡脖子（采购、技术开发依赖外部）→ Threat

## SWOT + TOWS

**SWOT 每条目标格式（强制）**：

```
- [来源: <框架名-子项>] 内容描述 [证据标签]
```

例：
```
- [来源: VRIO-超大组织通讯录] 2300 万家组织覆盖形成网络效应，转换成本极高 [公开数据]
- [来源: PEST-Political-双减] 教育场景受双减政策冲击 [公开数据]
- [来源: 价值链-营销与销售] 销售铁军规模与产能落后于飞书 [需验证]
```

**来源 → SWOT 桶位映射**：

| 上游框架 | → 进 SWOT 的哪个桶 |
|---|---|
| VRIO 持续/暂时优势 | S |
| 价值链 differentiator + 领先活动 | S |
| 7S 高对齐区域 | S |
| 7S 错配区域 | W |
| VRIO 竞争劣势 / 未利用优势 | W |
| 价值链 cost driver + 落后活动 | W |
| PEST + 方向 ≥ 3 强度 | O |
| 五力强度下降的方向（如新进入者壁垒变高 = 行业更稳） | O |
| PEST − 方向 ≥ 3 强度 | T |
| 五力强度上升（替代品/竞争加剧）| T |

**TOWS 矩阵（强制）**：

|              | 优势 (S) | 劣势 (W) |
|--------------|----------|----------|
| **机会 (O)** | **SO**：用 S 抓 O 的攻击策略 | **WO**：补 W 抓 O 的扭转策略 |
| **威胁 (T)** | **ST**：用 S 防 T 的防御策略 | **WT**：避险/收缩/退出策略 |

每象限至少 1 条具体策略，标注用到了哪条 S/W/O/T（如 "SO-1: 用 S2+S3 抓 O1"）。

**最终战略建议（1-3 句）**：从 TOWS 中挑出**最高优先级**的策略（一般是 SO 或 WT），用"应坚守 X + 加速 Y + 防范 Z"或"在 A 上聚焦 + 在 B 上止血 + 在 C 上观望"的句式收束。
