# Onboarding：首次调用引导

**触发条件**：使用者第一次调用 skill 时，`~/.company-diagnosis/user-profile.md` 不存在。

**绝对规则**：在 onboarding 完成前 **禁止开始任何六框架诊断**。退回引导。

---

## Step 1：解释这个 skill 需要什么

跟使用者说（中文，简短）：

> 这个 skill 给你做企业诊断时会锚定到 **你自己的商业路径**。我先问你 4 块基本信息，写到本地 `~/.company-diagnosis/user-profile.md`。这个文件会随你不断调研和自我迭代而更新，不会进任何公开仓库。
>
> 全部填完大约需要 10-15 分钟。中途可以跳过非必填项。

## Step 2：用 AskUserQuestion 分块收集

按 [user-profile-template.md](user-profile-template.md) 的字段，分 4 块收集。

### Block 1：身份定位（必填）

**问 2 件事**：
- 公司名 + 一句话业务（不超过 25 字，要能让"完全不知道你的人"立刻明白你做什么）
- 商业模式（你怎么赚钱：服务费 / 订阅 / 抽佣 / 项目制 / 广告 / SaaS / 混合，**要具体到收费方式**）

❌ 不要接受这种回答："我们是做 AI 创新的"——逼使用者落到"卖谁、收谁的钱、收多少"。

### Block 2：客户与阶段（必填）

**问 2 件事**：
- 目标客户画像 = 行业 + 规模 + 决策角色（例："年营收 3-30 亿的传统制造业老板"）
- 当前阶段 = `创意 / MVP / 早期 PMF / 规模化 / 成熟期` 五选一 + 关键证据（"已有 N 个付费客户" / "MRR 多少"）

### Block 3：核心商业纪律（必填，3-5 条）

**重点**：要"合同条款级"的具体表述，**不要废话和正确的口号**。

❌ 反例：「我们坚持高品质交付」「以客户为中心」「长期主义」
✅ 正例：
- 「首单必须纯咨询，不打包咨询 + 软件混合单」
- 「不接年营收 < 3000 万的客户」
- 「单客户咨询费 ≥ 15 万元起」
- 「不上架钉钉 / 飞书 / 企微的服务商市场」

如果使用者只能说出软口号，**追问到红线为止**："如果这条被违反了，你愿意为此放弃单子吗？"

### Block 4：路线图与不确定性（推荐填，但允许暂缺）

**问 3 件事**：
- 重点对标 / 参照公司：2-5 家 + 至少 1 家标记为"必看基准"
- 当前 12 个月路线图：3-5 个关键里程碑 + 时间窗（按季度）
- 关键不确定性：3-5 个让你睡不着的问题

这块允许"先填部分"，但要标 `[v0-暂缺]`，之后慢慢补。

## Step 3：写入文件

按 [user-profile-template.md](user-profile-template.md) 的格式，把 4 块信息组装成完整 markdown，写入：

```
~/.company-diagnosis/user-profile.md
```

第一行版本号 `v1`，最后更新日期 = 当天。

## Step 4：确认 + 进入诊断

跟使用者复述一遍 4 块要点，确认无误后开始六框架分析。

---

## 跳过 / 简化模式

如果使用者明确说"不要 onboarding，直接分析"：

1. 创建一个空壳 profile：
   ```markdown
   # 未命名使用者 商业路径规划（精简模式）

   **版本**：v0-skipped
   **最后更新**：YYYY-MM-DD

   ⚠️ 使用者跳过了 onboarding。第七节"对使用者商业路径的启示"将按通用模式输出，无法做字段级反哺建议。
   ```

2. 在最终报告**第七节顶部加红字提示**：
   > ⚠️ 使用者跳过了 onboarding，本节按通用模式输出。强烈建议补 onboarding 后重跑以获得字段级反哺建议。

3. 不要因为 skipped 就放弃归档到 `memory/`——后期补 onboarding 后这些历史调研也有意义。

---

## 后续迭代

`user-profile.md` 不是一次填完。**每次六框架诊断结束后第七节会给字段级更新建议**。使用者拍板后：
- 旧版本归档到 `~/.company-diagnosis/user-profile.history/<旧版本号>-<日期>.md`
- 新版本号 v_._ → v_.\_+1
- 顶部更新 "最后更新" 日期

记忆系统详见 [memory-spec.md](memory-spec.md)。
