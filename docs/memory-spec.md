# 记忆系统规范

## 总览

| 类型 | 位置 | 作用 | 生命周期 |
|---|---|---|---|
| **短期** | `~/.company-diagnosis/session-cache/current.md` | 本次会话临时上下文（当前公司、临时偏好、未完成节段） | 会话结束清空 |
| **长期** | `~/.company-diagnosis/memory/<公司>-YYYY-MM-DD.md` + `_index.md` | 完整调研归档 + 4 维 tag 索引 | 永久 |

## 长期记忆:单次调研归档(v3 改造:md + pdf 双份)

**路径**:
- Markdown 源:`~/.company-diagnosis/memory/<公司中文名>-YYYY-MM-DD.md`
- **PDF 可视化版**:`~/.company-diagnosis/memory/<公司中文名>-YYYY-MM-DD.pdf`(v3 强制,铁律 11)

**为什么要双份**:
- `.md`:LLM 直接写、可继续编辑、版本对比方便、跨公司检索靠它
- `.pdf`:可视化(Mermaid 渲染过的流程图 / 决策树 / 矩阵)+ 适合发给他人 review

**内容**:六框架诊断报告全文(按 [output-template.md](output-template.md) 输出)+ 序章方法论 + 第七节"对使用者商业路径的启示" + 附录。直接保存生成的完整报告,**不做摘要也不做压缩**——原始素材最值钱。

**生成 PDF**:用 [tools/md2pdf.ps1](tools/md2pdf.ps1)(Windows)或 [tools/md2pdf.sh](tools/md2pdf.sh)(macOS / Linux)。如果 PDF 转换失败,**红旗**,必须修复(铁律 11)。

**同公司多次调研**:新建文件 `<公司>-<YYYY-MM-DD>.md`(不覆盖旧的),并在 `_index.md` 标 v2 / v3。

## 长期记忆：索引文件 `_index.md`

**路径**：`~/.company-diagnosis/memory/_index.md`

**格式**：单张 markdown 表，每行一次调研。

```markdown
# 调研索引

| 公司 | 日期 | 行业/赛道 | 公司规模/阶段 | 商业模式 | 目标客户 | 关键洞察 (≤30字) | 报告 |
|---|---|---|---|---|---|---|---|
| 钉钉 | 2026-05-12 | SaaS-协同办公 | 大厂事业部/成熟 | PLG+SLG双轮+aPaaS | 大企业+SMB+政企 | 三角护城河：组织网络+信创交付+ISV生态 | [./钉钉-2026-05-12.md] |
| 悟空CRM | 2026-05-18 | SaaS-CRM | 创业/早期PMF | 订阅+开源版 | SMB销售团队 | 开源转付费转化率低，被钉钉宜搭蚕食 | [./悟空CRM-2026-05-18.md] |
```

**4 维 tag 列固定**（按已锁定的）：
1. 行业 / 赛道
2. 公司规模 / 生命周期
3. 商业模式
4. 目标客户画像

每个 tag 写 1-3 个关键词，用 `+` 或 `/` 分隔。

## 检索规则（新公司诊断前必跑）

```
拟分析公司 X：
  Step 1：估算 X 的 4 维 tag
  Step 2：Read ~/.company-diagnosis/memory/_index.md
  Step 3：找与 X 至少 2 维 tag 重合的历史记录
  Step 4：
    ├─ 有 → 在新报告头部加 "## 相关历史调研" 段：
    │       - 列每条相似记录：哪几维 tag 重合、关键洞察 1 句话、链接
    │       - 标"直接引用"还是"需重新验证"（数据 > 6 个月默认重新验证）
    └─ 无 → 跳过该段
  Step 5：进入完整 6 框架流程
```

**重合度判断**：

| 重合维度数 | 处理 |
|:-:|---|
| 4/4 | "高度相关"，可大量复用，但仍跑完整流程 |
| 3/4 | "强相关"，在 SWOT 阶段交叉验证 |
| 2/4 | "弱相关"，仅在第七节做对照参考 |
| 0-1/4 | 不算相关，跳过引用 |

## 归档触发(诊断结束后强制,v3 改造)

1. 把生成的完整报告写入 `memory/<公司>-<YYYY-MM-DD>.md`
2. **运行 PDF 转换**:
   - Windows: `tools\md2pdf.ps1 -InputFile <md> -OutputFile <pdf> -Title "<公司> 六框架诊断"`
   - macOS / Linux: `tools/md2pdf.sh <md> <pdf> "<公司> 六框架诊断"`
3. 检查 `<公司>-<YYYY-MM-DD>.pdf` 是否生成成功(铁律 11 自检)
4. 提取 4 维 tag + 不超过 30 字的"关键洞察"
5. 在 `_index.md` 表里 append 一行(行内同时给出 md 和 pdf 链接)
6. 在最终对话末尾告知使用者:归档位置(md + pdf) + index 已更新

**如果归档失败 → 红旗,必须修复**。任何"我没找到这个目录" / "权限被拒" / "PDF 生成失败"立刻报错,**不要默默跳过**。

## 短期记忆约定（可选）

`~/.company-diagnosis/session-cache/current.md` —— 跨多轮对话保持上下文(长会话场景):

```markdown
# 当前会话上下文

**开始时间**：YYYY-MM-DD HH:MM
**当前分析公司**：<公司名>
**进度**：已完成到第 N 节 / 共 7 节
**使用者临时偏好**：（本次会话中提到的）
- 偏好用中文报告
- 关心 AI Agent 维度
**待补数据点**：
- 需要补 ___ 的财报数据
- 需要补 ___ 的政策证据
```

每次新会话开始时检查这个文件：
- 如果存在且 < 24 小时 → 询问使用者"上次中途停在 X 公司第 N 节，要继续还是开新的？"
- 如果 ≥ 24 小时或不存在 → 清空 / 跳过

## 隐私边界

**绝对不进 GitHub**：
- ❌ `~/.company-diagnosis/` 下所有文件
- ❌ 报告归档文件中的具体客户名 / 营收 / 内部数字（如果使用者有标"敏感"）

**可以分享**（如果使用者愿意贡献）：
- ✅ 已脱敏的诊断报告作为社区案例
- ✅ 新发现的 LLM 失败模式（贡献回 baseline-test-results.md）
- ✅ 新发现的可信信源（贡献回 source-skills-registry.md）
