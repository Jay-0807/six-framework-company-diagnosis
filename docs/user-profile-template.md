# 商业路径规划模板

写入位置:`~/.company-diagnosis/user-profile.md`

复制下面整段,填字段。

---

```markdown
# <使用者公司名> 商业路径规划

> 这是一份**活文档**。每次六框架诊断结束后第七节会给字段级更新建议;旧版本会自动归档到 `user-profile.history/`。

**版本**:v1.0
**最后更新**:YYYY-MM-DD
**下次复盘建议**:YYYY-MM-DD(每季度 + 重大事件后)

---

- **行业**:<电商/SaaS/制造/服务/金融/...>

## 字段使用规则(不删,保留作为自检)

- **第零节"本次诊断聚焦问题"是第七节的输入**——每次诊断前更新,第七节会逐问回答
- 所有"未填"字段不阻断诊断,但会在第七节"对使用者商业路径的启示"中标"profile 数据不足,启示降级"
- 红线被诊断结论挑战时,第七节会显式 flag 让你拍板
- 路线图与发现冲突时,第七节给出"更新建议"
- 标 `[v0.5-推导]` 的红线在首次诊断后必须升级为 `[v1-用户确认]`
```

---

## 字段约束

| 字段 | 必填? | 期望长度 |
|---|:-:|---|
| **本次诊断聚焦问题** | ✅ (v3 新增) | 1-3 条,每条 ≤ 50 字 |
| 行业 | ✅ | 词条 |
| 公司名 | ✅ | 短 |
| 一句话业务 | ✅ | ≤ 25 字 |
| 商业模式 | ✅ | 1-3 句话(根据行业自适应模板) |
| 目标客户画像 | ✅ | 1-2 句 |
| 当前阶段 | ✅ | 词条 + 证据 |
| 核心商业纪律 | ✅ (可推导) | 3-5 条,每条 ≤ 30 字;允许 `[v0.5-推导]` |
| 重点对标 / 参照公司 | 推荐 | 2-5 家 |
| 12 个月路线图 | 推荐 | 4 季度 |
| 关键不确定性 | 推荐 | 3-5 条 |

**Block 0 + 1 + 2 + 3 是必填,少一项 onboarding 不算完成。Block 3 允许推导。**
**Block 4 可暂缺,标 `[v0-暂缺]`,但 6 个月内必须补全。**
