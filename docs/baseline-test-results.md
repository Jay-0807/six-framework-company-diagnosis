# Baseline 测试记录（RED 阶段）

写这个 skill 之前，按 writing-skills 的 TDD 铁律，先用 general-purpose subagent 跑了一次裸测，记录失败模式。本文件**不是给客户看的**，是给未来迭代 skill 的人留底。

## 测试条件

- **靶子**：钉钉（DingTalk, by Alibaba）
- **subagent 类型**：general-purpose（无 skill 加载）
- **指令**：用 6 框架（PEST/五力/7S/VRIO/价值链/SWOT）顺序分析，产出 Markdown 报告
- **额外限制**：不许 WebSearch，纯靠内置知识
- **目的**：观察 LLM 在没有 skill 引导时的天然失败模式

## 观察到的 7 类失败模式

### 1. 6 个框架孤岛输出
SWOT 节直接堆 S/W/O/T，不引用前 5 节任何具体发现。看不出"组织网络效应"这条 Strength 来自 VRIO 还是凭空生成。

### 2. VRIO 退化成"高/中/低"打分
baseline 输出原文：

> 通义大模型 + AI 协同能力 | 高 | 中 | 中（飞书、企微也有） | 强 | 竞争平价—暂时优势

问题：
- 用了"高/中/低"模糊评分，没走 V→R→I→O 顺序决策树
- "竞争平价—暂时优势"逻辑矛盾——VRIO 决策树要求二选一，不存在"既是又是"
- "稀缺"那列填"中"是没意义的——VRIO 的 R 是 Y/N，不是程度

### 3. 7S 只列 7 项，零对齐分析
baseline 给了一张 7 行表，每行 1 句话现状描述，完了。**完全没有 7×7 对齐矩阵**。7S 的核心洞察是检查 7 个 S 之间的对齐/错配，baseline 跳过了这步。

### 4. 价值链分类错乱、无毛利分析
原文 "进货后勤 / 研发投入" 把两个不同活动糊在一起。Porter 的 Inbound Logistics 是原料采购入库，研发投入应该归技术开发（support activity）。此外：
- 没有任何活动标 "占总成本 %"
- 没有 cost driver vs differentiator 区分
- 没有 vs 竞品对比

### 5. PEST + 5 Forces 不向 SWOT 传导
baseline 在 PEST 里提了"双减政策对教育场景冲击"，但 SWOT 的 Threats 段独立写"教育业务受政策冲击后增长乏力"，**两条没有任何显式连接**。无法验证 SWOT 的来源是否合理。

### 6. 全文零证据标签
所有数字（"2300 万+组织"、"7 亿+用户"、"5000+ ISV"、"政企客户"）都是裸数据，没有 [公开数据] / [推断] / [需验证] 任何标记。客户读了无法判断真假，也无法验证。

### 7. TOWS 矩阵完全缺失
baseline 的 SWOT 列完后直接跳到"战略建议（一句话收束）"，跳过了 TOWS 的 SO/WO/ST/WT 四象限策略推导。结论的一句话变成空中楼阁。

## skill 如何针对性堵这 7 个洞

| 失败模式 | SKILL.md 对应铁律 |
|---|---|
| 1 框架孤岛 | 铁律 2：SWOT 每条必须 `[来源: 框架-子项]` |
| 2 VRIO 高/中/低 | 铁律 3：V→R→I→O 决策树 + 禁用模糊评分 |
| 3 7S 无对齐 | 铁律 4：必须做 7×7 对齐矩阵 |
| 4 价值链泛化 | 铁律 5：Porter 9 活动 + 毛利贡献 + 类型 + vs 竞品 |
| 5 PEST→SWOT 断链 | 铁律 2 + 工作流里"每节末尾留 → 输入到 SWOT 的"凭据 |
| 6 零证据标签 | 铁律 6：四种标签 + 红旗"全篇 [公开数据] = 0 → 退回" |
| 7 缺 TOWS | 铁律 7：SWOT 不接 TOWS 是半成品 |

## Baseline 原文（保留备份）

完整 baseline 输出见 conversation history（钉钉报告全文）。要点摘录：

- PEST 4 维度都覆盖了，深度尚可，但缺时间维度和强度评分
- 五力 5 力都覆盖了，强度评级用了"高/中-高/中/低"，缺总吸引力分数
- 7S 7 项罗列完整但**完全无对齐矩阵**
- VRIO 表格用了 4 列但是"高/中/低"评分（不是 Y/N），结论分类混乱
- 价值链分类有错位（进货后勤糊了研发投入），无毛利贡献度
- SWOT 各 5 条左右覆盖完整，但**全部无来源引用**
- 结论一句话收束，**跳过 TOWS**
